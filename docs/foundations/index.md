# Foundations

Before the dossiers, three things must be fixed: a **taxonomy** (so we can place any
model in a shared coordinate system), a **method** (so every topic is documented
consistently for three audiences), and a **template** (so dossiers are comparable
and complete).

- **[Taxonomy of Policy Models](taxonomy.md)** — the coordinate system: the axes
  along which every policy model differs (optimization vs simulation, top-down vs
  bottom-up, equilibrium vs disequilibrium, deterministic vs stochastic, …).
- **[The Three-Track Method](three-track-method.md)** — why every topic is written
  as Scholar / Engineer / Architect, and how the tracks stay in sync.
- **[How to Read (and Write) a Dossier](../foundations/dossier-template.md)** — the
  canonical section list every model dossier must fill.

!!! tip "Reading order"
    Read the taxonomy once. Every dossier thereafter positions its model against
    those axes in its first paragraph, so you always know *what kind of thing* you
    are looking at before diving into equations.
